**updates in v2**
**Index <script> update removes issue of formulas not matching due to different order of elements**
Replaced the manual formula check with this inside checkAnswer() with:

```text
// Use accepted answer smiles to generate the expected formula automatically
const expectedMol = OCL.Molecule.fromSmiles(currentTask.answersSmiles[0]);
const expectedFormula = expectedMol.getMolecularFormula().formula;

const studentFormula = studentMol.getMolecularFormula().formula;
if (studentFormula !== expectedFormula) {
  showMsg(
    "Formula mismatch.\nYou drew: " +
    studentFormula +
    "\nExpected: " +
    expectedFormula
  );
  return;
}
```

So the below block was replaced block:

```text
const studentFormula = studentMol.getMolecularFormula().formula;
if (studentFormula !== currentTask.formula) {
  showMsg(
    "Formula mismatch.\nYou drew: " +
    studentFormula +
    "\nExpected: " +
    currentTask.formula
  );
  return;
}

```
with the new one above.

That means the expected formula is generated by OpenChemLib from answersSmiles[0], so it uses the same ordering as the student structure check.

**Have also changed to having OpenChemLib provide the displayed formula in the question card. So the formula field in tasks.js is defunct:**

Changed initializePage() from:
```text
document.getElementById("formula").textContent = currentTask.formula;
```
to:
```text
const expectedMol = OCL.Molecule.fromSmiles(currentTask.answersSmiles[0]);
document.getElementById("formula").textContent =
  expectedMol.getMolecularFormula().formula;
```
Now you no longer need to maintain a separate formula field at all I will live in task.js incase it ever needs to be brought back.
